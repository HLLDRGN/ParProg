## 3. Описание алгоритма

### 3.1. Стратегия распараллеливания

Используется метод блочного распределения строк матрицы **A** между процессами MPI. Матрица **B** транспонируется на нулевом процессе и рассылается всем процессам для повышения кэш-эффективности вычислений (последовательный доступ к памяти).

### 3.2. Схема коммуникаций

- `MPI_Bcast` — рассылка размера матрицы и транспонированной матрицы B всем процессам  
- `MPI_Scatterv` — распределение строк матрицы A между процессами  
- `MPI_Gatherv` — сбор вычисленных строк результирующей матрицы C  
- `MPI_Reduce` — сбор максимального времени выполнения (узкое место)

### 3.3. Замер времени

Время измеряется функцией `MPI_Wtime()` и охватывает только вычислительную фазу (без I/O и коммуникаций), что позволяет корректно оценить производительность вычислительного ядра.

## 5. Анализ результатов

### 5.1. Ускорение

- Максимальное ускорение: **4.53x** (матрица 2000×2000)
- Ускорение стабильно выше **3.38x** для всех размеров ≥ 400×400
- Для матрицы 200×200 ускорение составило лишь **2.14x** — накладные расходы на коммуникации MPI превышают выигрыш от параллелизма при малом объёме вычислений

### 5.2. Эффективность

- Высокая эффективность для средних матриц (400–1200): **90–100%**
- Значение > 100% для матрицы 2000×2000 объясняется эффектами кэширования при разбиении данных на блоки
- Для малой матрицы (200×200) эффективность снижается до **53.5%**

### 5.3. Производительность (GFLOPS)

- Пик производительности: **1.61 GFLOPS** (матрица 400×400)
- Снижение GFLOPS для матриц > 400 связано с ростом числа кэш-промахов при увеличении рабочего набора данных
- Для матрицы 2000×2000: **0.69 GFLOPS**
